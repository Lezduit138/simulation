SJF — Key Concepts

AlgorithmWhen the CPU is free, select the ready process with the smallest burst time. Ties are broken by arrival time.
Non-preemptiveA running process is never interrupted. Even if a shorter job arrives mid-execution, it must wait until the CPU is free.
Optimal AWTSJF gives the minimum average waiting time among all non-preemptive scheduling algorithms for a given set of processes.
Waiting timeStart time − Arrival time. Short processes are favoured, so long processes may wait significantly longer.
StarvationLong processes can wait indefinitely if a continuous stream of short jobs keeps arriving — the main weakness of SJF.
vs FCFSSJF requires knowledge of burst times in advance; FCFS does not. SJF minimises average waiting time; FCFS is simpler but can cause the convoy effect.
